# CoreSdkProvider

The `<CoreSdkProvider>` is the component that is in charge of creating an sdk client by using the TokenProvider context.

<span type="warning" title="Dependencies">
It must be rendered as a children of the `<TokenProvider>` component.
</span>

<Source
  language="tsx"
  dark
  code={`
<TokenProvider
  kind="orders"
  appSlug="orders"
  domain={window.clAppConfig.domain}
  reauthenticateOnInvalidAuth={!isDev}
  devMode={isDev}
>
  <CoreSdkProvider>
    <div>app content</div>
  </CoreSdkProvider>
</TokenProvider>
`}
/>



## useCoreSdkProvider

This hook returns an instance of the [@commercelayer/sdk](https://github.com/commercelayer/commercelayer-sdk) client.

<span type="info">
Whenever possible we encourage you to use the [`useCoreApi`](#usecoreapi) hook.

The `useCoreSdkProvider` hook is mostly used to perform **create**/**update**/**delete** operations.
</span>

## useCoreApi

This hook performs api request base on [@commercelayer/sdk](https://github.com/commercelayer/commercelayer-sdk) and [swr](https://swr.vercel.app/).

Suppose that you need to get the order with ID `1234`.

<Source
  language="ts"
  dark
  code={`
// using our sdk
const order = await client.orders.retrieve('1234')

// using \`useCoreApi\`
const {
  data: order,
  isLoading
} = useCoreApi('orders', 'retrieve', ['1234'])
`} />
